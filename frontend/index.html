RAG Architecture

Retrieval‑Augmented Generation, built for academia

1. Document Ingestion

PDFs, lecture notes, ArXiv papers are parsed and chunked.

Chunk size: 512 tokens • Overlap: 50 tokens

2. Embedding

Text chunks → dense vectors via all‑MiniLM‑L6‑v2.

384‑dim embeddings • Batch size 32

3. Vector Storage

FAISS index with IVF + HNSW for 10M+ vectors.

nlist=4096 • nprobe=10 • <20ms latency

4. Query & Generate

User query → retrieve top‑k → LLM synthesises answer.

k=5 • Llama 3 8B • Citations included
