RAG Architecture

Retrieval-Augmented Generation, built for academia

1. Document Ingestion

PDFs are parsed and chunked into 500-character segments.

Chunk size: 500 chars · Overlap: 50

2. Embedding

Text chunks → 384-dim dense vectors via all-MiniLM-L6-v2.

384-dim · L2 normalised

3. Vector Storage

FAISS IndexFlatIP — fast inner-product search, persisted to disk.

<20ms latency · saved to data/

4. Query & Generate

Top-5 passages + Llama 3 → grounded, cited answer.

k=5 · Groq API · Llama-3.1-8b-instant
