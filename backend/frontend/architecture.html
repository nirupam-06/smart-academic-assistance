System Design

Retrieval-Augmented Generation

From raw PDFs to cited answers — a complete RAG pipeline built for academia.

01

Ingestion

PDFs are opened with PyMuPDF, text extracted page by page, then split into 500-character chunks with 50-character overlap to preserve context at boundaries.

PyMuPDF · 500 char chunks · 50 overlap
02

Embedding

Each chunk is hashed into a 384-dimensional float32 vector using a lightweight word-frequency hash approach — no GPU, no large model download, Railway-compatible.

384-dim · L2 normalised · pure Python
03

Vector Storage

FAISS IndexFlatIP stores all embeddings in memory and saves to disk after each upload. On startup, the index is loaded from disk — so your documents persist across restarts.

FAISS IndexFlatIP · data/faiss.index
04

Retrieval

The user's query is embedded with the same function, then FAISS performs inner-product search to find the top-5 most relevant chunks from all indexed documents.

k=5 · cosine similarity · <20ms
05

Generation

The 5 retrieved passages are formatted as context and sent to Llama 3.1 8B via Groq API. The model synthesises a grounded answer with source filenames cited.

Llama-3.1-8b-instant · Groq API · max_tokens 1024
Specs
Embedding Hash-384 Pure Python, no GPU
Vector Index FAISS Flat Inner-product search
Retriever k = 5 Top-5 passages
Generator Llama 3.1 8B Via Groq API
